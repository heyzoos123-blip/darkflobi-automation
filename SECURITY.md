# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |

## Reporting a Vulnerability

We take security seriously at Darkflobi. If you discover a security vulnerability, please report it responsibly.

### How to Report

1. **DO NOT** create a public GitHub issue for security vulnerabilities
2. Email security concerns to the team via Twitter DM [@darkflobi](https://twitter.com/darkflobi)
3. Include as much detail as possible:
   - Type of vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

### What to Expect

- **Acknowledgment:** Within 48 hours
- **Status Update:** Within 7 days
- **Resolution Timeline:** Depends on severity

### Scope

This security policy applies to:
- The main darkflobi-automation repository
- Official integrations and plugins
- API endpoints

### Out of Scope

- Third-party dependencies (report to maintainers)
- Social engineering attacks
- Denial of service attacks

## Security Best Practices

When using Darkflobi:
- Never commit API keys or secrets
- Use environment variables for sensitive data
- Keep dependencies updated
- Review code before running
